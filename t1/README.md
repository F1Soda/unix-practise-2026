# Task 1

## Запуск

```sh
./runme.sh
```

## Ожидаемый результат

1. Все файлы A, B, C, D должны иметь одинаковый логический размер
2. Дырявый файл B после работы скрипта sparser над файлом A должна дать меньший физический размер
3. сжатые файл A.gz и дырявый файл B.gz должны иметь одинаковый размер
4. Если передать на вход программы sparser расжатый файл B.gz то результирующий файл C должен по блокам
   совпадать с B. Фактический это то же самое что вызов sparser B A, так как расжатый файл, который ранее был дырявым
   становится обычным и теряет свои свойства
5. При использовании нестандартного размера буфера для sparser может получится так, что новый дырявый файл D будет
   иметь больше блоков чем дырявый файл B

   > Допустим размер блока в 4 кбайт, а размер буфера будет 4000 байт Исходный файл размером 8 кбайт + 1 байт и содержит данные так:
   > - от 0 до 3998 стоят нули,
   > - с 3999 до 4096 стоят единицы,
   > - а с 4097 до 8192 стоят нули,
   > - а на конце в 8193 стоит еденица
   >
   > По идее на данные нужно 2 блока, но из-за нестандартного размера буфера он первый буфер зачитает как с данными,
   > потому что увидит 1 на 3999 позиции и запишет 000000...1
   >
   > Затем начнет читать дальше, увидит что там тоже есть данные и снова запишет 111...11000..000
   >
   > и затем последний блок запишет как обычно с данными
   >
   > Получится 3 блока

   Вывод: при создании дырявых файлов важно учитывать размер блоков в ФС системы
